Case study · Strona WWW

Zakres prac — LED.digitais.pl

Projekt: Aplikacja sprzedażowa dla ekranów LED | https://led.digitais.pl

Typ projektu Aplikacja webowa full-stack
Technologie React, TypeScript, Node.js, PostgreSQL
Status Wdrożona produkcyjnie

Projekt obejmował wykonanie kompletnej aplikacji webowej full-stack: landing page sprzedażowego, interaktywnego kalkulatora ROI, formularza pozyskiwania leadów, panelu administracyjnego CRM, backendu API oraz bazy danych.

Co zostało zbudowane

1. Landing page sprzedażowy PL

Zaprojektowałem i wdrożyłem polskojęzyczną stronę sprzedażową pozycjonującą transparentne ekrany LED jako źródło dodatkowego dochodu dla właścicieli sklepów w Warszawie.

  • Sekcja hero z kartą „Potencjalny zysk” synchronizowaną na żywo z kalkulatorem ROI.
  • Interaktywny kalkulator ROI działający po stronie klienta.
  • Kalkulacja uwzględniająca typ ekranu, dzielnicę Warszawy, liczbę reklamodawców, własną stawkę i koszt energii.
  • Interaktywna mapa dzielnic Warszawy połączona z kalkulatorem.
  • Sekcje korzyści, gwarancji 90-dniowej, specyfikacji technicznej, procesu wdrożenia i FAQ.
  • Formularz pozyskiwania leadów z walidacją po stronie serwera i zgodami RODO.
  • Tabela referencyjna rat leasingowych: 48 miesięcy, 25% wykupu, 0% wpłaty własnej.

2. Panel administracyjny /admin

Powstał kompletny system CRM dla zespołu sprzedażowego.

  • Bezpieczne logowanie i wylogowanie administratora.
  • Sesja cookie i hasło hashowane SHA-256.
  • Dashboard z wykresami: trendy leadów, rozkład statusów i mapa dzielnic.
  • Lista leadów z filtrowaniem po statusie i dzielnicy, wyszukiwarką oraz paginacją.
  • Widok szczegółowy leada z edycją statusu inline i notatkami.
  • Usuwanie pojedynczego leada z potwierdzeniem.
  • Masowe usuwanie zaznaczonych leadów z potwierdzeniem.
  • Edytor stawek dla dzielnic: minimalna i maksymalna stawka reklamodawcy oraz aktywność dzielnicy.
  • Eksport leadów do CSV jednym kliknięciem.

3. Backend API REST/OpenAPI

  • Kontrakt API-first: OpenAPI 3.0.
  • Automatyczny codegen hooków React Query i schematów Zod.
  • Endpointy: leads CRUD, eksport CSV, zarządzanie dzielnicami i autoryzacja.
  • Walidacja wejść i wyjść przez Zod.
  • Obsługa sekcji statycznych SPA oraz API na jednym serwerze Express 5.

4. Baza danych

  • PostgreSQL + Drizzle ORM.
  • Schemat danych: leads, districts, users.
  • Dzielnice Warszawy seedowane przy starcie z pliku konfiguracyjnego.
  • Dzielnice edytowalne z poziomu panelu administratora.

5. Responsywność desktop/mobile

  • W pełni responsywny layout na urządzeniach mobilnych i desktopowych.
  • Przyciski wyboru dzielnicy: siatka 2-kolumnowa mobile oraz flex z zawijaniem desktop.
  • Navbar panelu admina z układem dostosowanym do mobile.
  • Karta „Potencjalny zysk” czytelna na wszystkich rozdzielczościach.

6. Optymalizacja wydajności

  • Lighthouse: Mobile 95 / Desktop 100 / SEO 100 / Best Practices 100.
  • Czcionki systemowe — bez opóźnienia ładowania fontów.
  • Recharts ładowany lazy i bez blokowania renderowania.
  • Kompresja gzip na serwerze.
  • Agresywne nagłówki cache dla zasobów statycznych.
Efekt: strona została wdrożona pod domeną https://led.digitais.pl i jest gotowa do pozyskiwania leadów od właścicieli sklepów w Warszawie. Zespół sprzedażowy ma dostęp do panelu zarządzania leadami bez potrzeby korzystania z zewnętrznych narzędzi CRM.
